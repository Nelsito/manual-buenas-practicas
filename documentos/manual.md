# 📘 Manual de Organización y Buenas Prácticas
## 1. Gestión del Tiempo y Metodologías Ágiles
El colapso a final de semestre se evita adoptando marcos de trabajo como **Scrum**. 

### 1.1. Ciclo de Trabajo Recomendado:
* **Planificación:** Definir el Product Backlog.
* **Ejecución:** Dividir el trabajo en Sprints.
* **Revisión:** Evaluar lo avanzado.

## 2. Políticas de Comunicación Interna
Para el éxito del equipo, la comunicación debe ser asíncrona pero constante. Se desaconseja el uso exclusivo de WhatsApp para decisiones técnicas. Todo acuerdo debe quedar documentado en las minutas del repositorio o en los comentarios de los Pull Requests.

## 3. Normativas de Formato Académico
El rigor académico en la UVM exige estandarización.
* **Fuente:** Times New Roman 12pt o Arial 11pt.
* **Interlineado:** 2.0 (Doble espacio).
* **Alineación:** Izquierda (sin justificar).

## 4. Ecosistema Colaborativo y Citas APA
El equipo debe centralizar su trabajo:

| Tarea | Herramienta | Propósito |
| :--- | :--- | :--- |
| **Versiones** | Git / GitHub | Historial de código |
| **Redacción** | Google Docs | Edición en tiempo real |

**Ejemplo de Citación (Libro):** > Apellido, A. A. (Año). *Título del libro*. Editorial.
